Purpose of This Page

This page provides the authoritative status of the Public Baseline Declaration, including current version, release date, and scope. The Public Baseline establishes minimum child-safe conditions for AI use and remains freely accessible and referenceable.

What the Public Baseline Covers

  • Pre-interaction declaration of role, limits, and responsibility
  • Non-overridable child safety constraints
  • Explicit exclusions (medical, legal, psychological, behavioral influence)
  • Model-agnostic applicability

What the Public Baseline Does Not Provide

  • Institutional implementation wording
  • Staff or student operational language
  • Liability boundary definitions
  • Audit or inspection-ready controls

For institutional reliance, see the AI Child Use — School Implementation Standard.

Update & Reliance Notice

Updates to the Public Baseline are published on this page. Minor clarifications may be issued as v1.x. Material changes will be issued as v2.0+. Previous versions remain archived and referenceable. No individual notifications are issued.

Any uncertainty about whether Child Mode is active requires the Child Mode declaration to be reloaded.

Version History

  • v1.0 — 2026-01-08: Initial public release.

Relationship to the School Implementation Standard

The Public Baseline defines the minimum safety floor. The School Implementation Standard operationalises this baseline for formal school adoption and institutional reliance.

Use & Attribution

This declaration may be used and referenced freely, provided it is reproduced as written and not misrepresented. Where used in institutional settings, reference should include the version and this status page link.
